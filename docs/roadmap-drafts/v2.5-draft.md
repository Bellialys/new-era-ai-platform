## Что входит в v2.5

### 1. API Key Management

**Хранение — HMAC-SHA256 (не bcrypt):**
bcrypt слишком ресурсоёмок для верификации при каждом HTTP-запросе.
Для API-ключей (высокая энтропия, длинные) стандарт индустрии — HMAC-SHA256:
```typescript
const keyHash = crypto.createHmac('sha256', process.env.API_KEY_SECRET)
  .update(rawKey).digest('hex');
```
Верификация — микросекунды вместо ~100ms у bcrypt.

**Префиксы ключей:**
- `ne_sk_live_...` — production-ключ (реальные LLM-вызовы, биллинг)
- `ne_sk_test_...` — Test Mode (детерминированные моки, стоимость $0)

**Два активных ключа — Zero-downtime rotation:**
Таблица `api_keys` позволяет иметь 2 активных ключа одновременно.
«Rotate» в UI создаёт новый ключ и запускает 24-часовой период перекрытия —
старый ключ остаётся рабочим до `rotation_expires_at`.

```sql
CREATE TABLE api_keys (
  id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id             UUID NOT NULL REFERENCES auth.users(id),
  key_hash            TEXT NOT NULL UNIQUE,   -- HMAC-SHA256
  key_prefix          TEXT NOT NULL,          -- первые 8 символов для отображения
  name                TEXT NOT NULL,
  scopes              TEXT[] NOT NULL DEFAULT '{}',
  expires_at          TIMESTAMPTZ NOT NULL,   -- обязательный expiry
  rotation_expires_at TIMESTAMPTZ,            -- grace период при ротации
  last_used_at        TIMESTAMPTZ,
  revoked_at          TIMESTAMPTZ,
  created_at          TIMESTAMPTZ DEFAULT now()
);

-- Аудит операций с ключами
CREATE TABLE api_key_audit (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  api_key_id  UUID REFERENCES api_keys(id),
  user_id     UUID NOT NULL,
  action      TEXT NOT NULL,  -- 'created', 'revoked', 'rotated', 'auth_failed'
  ip_address  TEXT,
  user_agent  TEXT,
  created_at  TIMESTAMPTZ DEFAULT now()
);
```

**Scopes:**
- `read:models` — GET /api/v1/models
- `write:compare` — POST /api/v1/compare
- `read:usage` — GET /api/v1/usage
- `write:webhooks` — управление webhook endpoints

**Лимиты по плану:** `free` — 1 ключ, `pro` — 10, `enterprise` — без лимита.
Создание ключей: rate limit 5/час.

**Expiry уведомления:**
За 7 дней до истечения → email + баннер в Developer Dashboard.

### 2. Публичный REST API `/api/v1/`

**Аутентификация:**
```
Authorization: Bearer ne_sk_live_...
```
Middleware верифицирует HMAC-SHA256 за O(1).

**Rate Limits изолированы по api_key_id:**
Скомпрометированный ключ не исчерпывает UI-квоту пользователя.
Отдельные счётчики в Upstash Redis: `rl:key:{api_key_id}:{endpoint}`.

**Заголовки ответа:**
```
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 47
X-RateLimit-Reset: 1751234567
Retry-After: 13          (только при 429)
X-API-Version: v1
```

**Эндпоинты:**

`POST /api/v1/compare`
```json
// Request
{
  "prompt": "Объясни квантовую запутанность",
  "models": ["gpt-4o", "claude-3-5-sonnet"],
  "maxTokens": 1000
}
// Response
{
  "id": "cmp_uuid",
  "responses": [
    { "model": "gpt-4o", "text": "...", "latencyMs": 1240, "costUsd": 0.023 }
  ],
  "totalCostUsd": 0.041,
  "createdAt": "2026-07-01T10:00:00Z"
}
```

**Idempotency-Key:**
```
Idempotency-Key: client-generated-uuid
```
Кэш в Redis с TTL 24 часа. Повторный запрос с тем же ключом возвращает
закэшированный ответ без повторного LLM-вызова и биллинга.

`GET /api/v1/models`
Список доступных моделей с ценами. Публичный (auth не требуется).

`GET /api/v1/usage?cursor=...&limit=50&from=2026-06-01&to=2026-07-01`
**Cursor-based pagination** (не OFFSET/LIMIT):
```json
{
  "data": [...],
  "nextCursor": "eyJpZCI6InV1aWQifQ==",
  "hasMore": true
}
```

`GET /api/v1/leaderboard?cursor=...&limit=20`
Cursor-based pagination.

### 3. Test Mode

Ключи `ne_sk_test_...` не делают реальных вызовов к LLM-провайдерам.
Возвращают детерминированные моки:
```json
{
  "id": "cmp_test_fixed",
  "responses": [
    { "model": "gpt-4o", "text": "[TEST] This is a mock response.", "latencyMs": 50, "costUsd": 0 }
  ],
  "totalCostUsd": 0,
  "_test": true
}
```
Идеально для CI/CD тестов. Биллинг не происходит.

### 4. Webhooks

**Регистрация endpoint:**
`POST /api/v1/webhooks` (scope: `write:webhooks`):
```json
{ "url": "https://example.com/webhook", "events": ["compare.completed", "balance.low"], "secret": "user-provided-secret" }
```

**SSRF защита:**
Перед регистрацией и перед каждой доставкой — резолвинг домена и проверка IP.
Блокируем: RFC 1918 (10.x, 172.16-31.x, 192.168.x), loopback (127.x), link-local (169.254.x).

**Подпись с replay-атак защитой:**
```
X-Signature: sha256=hmac(secret, timestamp + "." + body)
X-Timestamp: 1751234567
```
Клиент проверяет что `|now - timestamp| < 300` (5 минут).

**События:**
- `compare.completed` — результат /api/v1/compare
- `balance.low` — баланс ниже auto-recharge threshold
- `subscription.changed` — смена плана

**Retry:** exponential backoff, max 5 попыток (1m → 5m → 30m → 2h → 24h).
Таймаут ожидания ответа от endpoint: **5 секунд**.

```sql
CREATE TABLE webhook_endpoints (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL,
  url         TEXT NOT NULL,
  events      TEXT[] NOT NULL,
  secret_hash TEXT NOT NULL,
  enabled     BOOLEAN DEFAULT TRUE,
  created_at  TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE webhook_deliveries (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  endpoint_id       UUID REFERENCES webhook_endpoints(id),
  event_type        TEXT NOT NULL,
  payload           JSONB NOT NULL,
  idempotency_key   TEXT NOT NULL UNIQUE,
  status_code       INTEGER,
  response_body     TEXT,
  attempt_count     INTEGER DEFAULT 0,
  next_retry_at     TIMESTAMPTZ,
  delivered_at      TIMESTAMPTZ,
  created_at        TIMESTAMPTZ DEFAULT now()
);
```

### 5. TypeScript SDK

npm-пакет `@new-era/sdk`:
```typescript
import { NewEra, BudgetExceededError, RateLimitError } from '@new-era/sdk';

const client = new NewEra({ apiKey: 'ne_sk_live_...' });

try {
  const result = await client.compare({
    prompt: 'Explain quantum entanglement',
    models: ['gpt-4o', 'claude-3-5-sonnet'],
  }, { idempotencyKey: 'my-request-id' });
} catch (err) {
  if (err instanceof BudgetExceededError) { /* handle */ }
  if (err instanceof RateLimitError) { /* retry after err.retryAfter */ }
}
```

Типизированные классы ошибок: `BudgetExceededError`, `RateLimitError`,
`AuthError`, `InvalidScopeError`, `ApiError`.

Автогенерация типов из OpenAPI схемы.
Tree-shakable, работает в Node.js, Deno, Bun.

### 8. Feature Flags

```sql
CREATE TABLE feature_flags (
  key              TEXT PRIMARY KEY,
  enabled          BOOLEAN DEFAULT FALSE,
  rollout_percent  INTEGER DEFAULT 0 CHECK (rollout_percent BETWEEN 0 AND 100),
  allowed_user_ids UUID[] DEFAULT '{}',
  updated_by       UUID,
  updated_at       TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE feature_flag_history (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  flag_key   TEXT NOT NULL,
  old_value  JSONB,
  new_value  JSONB,
  changed_by UUID NOT NULL,
  changed_at TIMESTAMPTZ DEFAULT now()
);
```

Кэш в памяти + `pg_notify` инвалидация (та же логика что `plan_entitlements` из v2.4).
Управление через `/admin/flags` и REST endpoint для CI/CD автоматизации.

### 9. S3 бэкап billing_events

Vercel Cron раз в сутки (03:00 UTC):
1. Экспорт `billing_events` за вчера → JSONL с SHA-256 чек-суммой
2. Upload в Cloudflare R2 (или S3) с шифрованием `AES-256`
3. Запись в таблицу `backup_log` (date, file_key, checksum, size_bytes, status)
4. Если бэкап не создан или checksum не совпадает → алерт администратору
