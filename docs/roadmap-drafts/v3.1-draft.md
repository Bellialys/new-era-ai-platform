## Что входит в v3.1

### 1. Regional Migration API (EU ↔ US)

**Паттерн Strangler Fig — обязателен из-за изоляции регионов:**

PostgreSQL не поддерживает Foreign Keys между физически разными базами (EU и US —
изолированные Supabase проекты). При наивном переносе ID-последовательности
рассинхронизируются. Решение — экспорт в нейтральный формат + импорт с новыми ID
+ маппинг-таблица.

```sql
CREATE TABLE org_region_migrations (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id          UUID NOT NULL REFERENCES organizations(id),
  from_region     TEXT NOT NULL CHECK (from_region IN ('eu', 'us')),
  to_region       TEXT NOT NULL CHECK (to_region IN ('eu', 'us')),
  status          TEXT DEFAULT 'pending_approval'
                  CHECK (status IN (
                    'pending_approval', 'dry_run', 'dry_run_complete',
                    'approved', 'in_progress', 'cdc_sync',
                    'checksum_validation', 'completed', 'failed', 'rolled_back'
                  )),
  -- Maker-Checker с 2FA
  initiated_by    UUID NOT NULL,
  initiated_2fa_at TIMESTAMPTZ,     -- timestamp подтверждения 2FA initiator
  approved_by     UUID,
  approved_2fa_at TIMESTAMPTZ,      -- timestamp подтверждения 2FA approver
  approval_comment TEXT,
  -- Метрики выполнения
  rows_exported   BIGINT,
  rows_imported   BIGINT,
  checksum_source TEXT,
  checksum_target TEXT,
  -- Временные метки
  dry_run_at      TIMESTAMPTZ,
  started_at      TIMESTAMPTZ,
  completed_at    TIMESTAMPTZ,
  shredding_scheduled_at TIMESTAMPTZ,  -- +30 дней grace period
  shredded_at     TIMESTAMPTZ,
  -- Ошибки и откат
  error           TEXT,
  rollback_reason TEXT,
  created_at      TIMESTAMPTZ DEFAULT now()
);

-- Маппинг старых ID → новых (для восстановления связей после миграции)
CREATE TABLE migration_id_mappings (
  migration_id    UUID NOT NULL REFERENCES org_region_migrations(id),
  table_name      TEXT NOT NULL,
  old_id          UUID NOT NULL,
  new_id          UUID NOT NULL,
  PRIMARY KEY (migration_id, table_name, old_id)
);

-- CDC: все изменения во время миграции
CREATE TABLE migration_changes_log (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  migration_id    UUID NOT NULL REFERENCES org_region_migrations(id),
  table_name      TEXT NOT NULL,
  operation       TEXT NOT NULL CHECK (operation IN ('INSERT', 'UPDATE', 'DELETE')),
  record_id       UUID NOT NULL,
  payload         JSONB NOT NULL,
  captured_at     TIMESTAMPTZ DEFAULT now()
);

-- Полный аудит каждого шага
CREATE TABLE org_region_migrations_audit (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  migration_id    UUID NOT NULL REFERENCES org_region_migrations(id),
  actor_id        UUID NOT NULL,
  action          TEXT NOT NULL,
    -- 'dry_run_started', 'dry_run_completed', 'approved', 'migration_started',
    -- 'cdc_sync_started', 'checksum_validated', 'jwt_switched',
    -- 'completed', 'rollback_initiated', 'shredding_completed'
  detail          JSONB,
  created_at      TIMESTAMPTZ DEFAULT now()
);
```

**Алгоритм Zero-Downtime Migration:**

```
Фаза 0: Dry-run (обязательно перед реальной миграцией)
  1. Копируем данные org в target регион (тестовый tenant)
  2. Верифицируем checksums
  3. Симулируем JWT переключение (в staging)
  4. Автоочистка тестовых данных
  5. Отчёт: rows_exported, rows_imported, checksum match

Фаза 1: Подготовка
  1. platform_readonly = TRUE только для данной org (не платформы целиком)
  2. Начинаем CDC через Supabase Realtime → migration_changes_log

Фаза 2: Миграция
  1. Export всех данных org → Parquet/NDJSON в R2
  2. Import в target регион с НОВЫМИ ID
  3. Записываем migration_id_mappings (old_id → new_id)

Фаза 3: Инкрементальный CDC sync
  1. Применяем накопленные migration_changes_log к target
  2. Повторяем до lag < 1 секунды

Фаза 4: Переключение
  1. Верификация checksums (source vs target)
  2. При mismatch → автоматический rollback
  3. При успехе → обновляем global_accounts.data_region
  4. platform_readonly = FALSE
  5. JWT пользователей начинают указывать на новый регион

Фаза 5: Grace period (30 дней)
  1. Исходные данные остаются в source регионе (read-only)
  2. После 30 дней → Crypto-Shredding DEK source региона
```

**Maker-Checker с 2FA:**
- Initiator инициирует → подтверждает своим TOTP
- Approver (второй суперадмин) получает email → подтверждает своим TOTP + комментарий
- Без обеих подписей → миграция не запускается

**Migration lifecycle UI** (`/admin/global/migrations`):
Статус каждой миграции, кнопки Cancel / Retry / Force-Shred / Validate-Checksums.

### 2. Client-Side BYOK с trace_token

Для Enterprise с собственными ключами у провайдеров — избегает проблему Shared IP.

```sql
-- Одноразовые trace_token
CREATE TABLE byok_trace_tokens (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id          UUID NOT NULL REFERENCES organizations(id),
  user_id         UUID NOT NULL,
  provider_id     UUID NOT NULL REFERENCES marketplace_providers(id),
  model_key       TEXT NOT NULL,
  max_tokens      INTEGER NOT NULL,
  estimated_cost_usd NUMERIC(10,6) NOT NULL,
  reserved_amount_usd NUMERIC(10,6) NOT NULL,  -- заблокировано на балансе
  jti             TEXT NOT NULL UNIQUE,          -- JWT ID для одноразовости
  allowed_ip      INET,
  expires_at      TIMESTAMPTZ NOT NULL,          -- now() + 30 seconds
  used_at         TIMESTAMPTZ,                   -- NULL = не использован
  status          TEXT DEFAULT 'issued'
                  CHECK (status IN ('issued', 'used', 'expired', 'callback_received', 'pending_verification')),
  created_at      TIMESTAMPTZ DEFAULT now()
);

-- Callback'и от провайдеров (idempotent)
CREATE TABLE byok_callbacks (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  provider_id     UUID NOT NULL REFERENCES marketplace_providers(id),
  event_id        TEXT NOT NULL,            -- уникальный ID от провайдера
  trace_token_jti TEXT NOT NULL REFERENCES byok_trace_tokens(jti),
  actual_tokens   INTEGER,
  actual_cost_usd NUMERIC(10,6),
  provider_signature TEXT,                 -- HMAC подпись провайдера
  processed_at    TIMESTAMPTZ,
  created_at      TIMESTAMPTZ DEFAULT now(),

  UNIQUE (provider_id, event_id)           -- идемпотентность
);
```

**Безопасность trace_token (JWT):**
```typescript
// Payload trace_token (подписан приватным ключом EdDSA)
{
  "jti": "unique-uuid",          // JWT ID — проверяем что не использован
  "iss": "https://api.domain.com",
  "sub": "org_uuid:user_uuid",
  "aud": "provider_slug",        // только для конкретного провайдера
  "model": "cohere-command-r",
  "max_tokens": 10000,
  "estimated_cost_usd": "0.05",
  "allowed_ip": "203.0.113.42",  // IP клиента
  "exp": 1751234597              // now() + 30 секунд
}
```

**JWKS endpoint:** `GET /.well-known/jwks.json` — провайдеры верифицируют подпись.

**Pre-Authorization:**
```sql
-- При выдаче trace_token резервируем estimated_cost
SELECT pg_advisory_xact_lock(hashtext(org_id::text));
UPDATE organizations SET balance_usd = balance_usd - estimated_cost
WHERE id = org_id AND balance_usd >= estimated_cost;
-- Если insufficient → 402, trace_token не выдаётся

-- При получении callback → освобождаем резерв и списываем факт
UPDATE organizations SET balance_usd = balance_usd + (estimated_cost - actual_cost)
WHERE id = org_id;
```

**Таймаут 5 минут:** Inngest job проверяет `byok_trace_tokens WHERE status = 'used' AND used_at < now() - INTERVAL '5 minutes'` → `status = 'pending_verification'` → алерт + автоосвобождение резерва.

**BYOK Sandbox:**
`X-BYOK-Sandbox: true` → mock callback, нет Platform Fee, нет влияния на аналитику.

### 3. GraphQL API (Enterprise)

`POST /api/graphql` — только для Enterprise плана.

**N+1 — DataLoader:**
```typescript
// Без DataLoader: N запросов к БД для N пользователей
// С DataLoader: 1 батч-запрос
const memberLoader = new DataLoader(async (userIds) => {
  const members = await db.select().from(organizationMembers)
    .where(inArray(organizationMembers.userId, userIds));
  return userIds.map(id => members.find(m => m.userId === id));
});
```

**Persisted Queries (Automatic Persisted Queries):**
```bash
# Разработчик регистрирует query через CLI
npx @new-era/cli graphql register ./queries/compare.graphql
# → возвращает hash: sha256:abc123...

# Production принимает только
POST /api/graphql
{ "extensions": { "persistedQuery": { "version": 1, "sha256Hash": "abc123..." } } }
```

Произвольные query string в production → 400 Bad Request.
Исключение: authenticated admin для разработки.

**Complexity Limits:**
```typescript
// Каждое поле имеет complexity score
// Вложенность добавляет множитель
// Сложный запрос: members(100) { usageEvents(50) } = 5000 → reject
const complexityLimit = {
  enterprise: 1000,
  pro: 300,
};
```

**Интроспекция:** отключена в production. Доступна только для `role = 'admin'`.

**@cacheControl для read-only:**
```graphql
type Query {
  models: [Model!]! @cacheControl(maxAge: 300)    # 5 минут
  usage(from: DateTime!, to: DateTime!): Usage     # no cache (per-user)
}
```

**GraphQL Playground** (`/api/graphql/playground`):
- Отображает complexity score запроса в реальном времени
- Estimated cost до выполнения
- Только для authenticated users

**URL versioning:** `/api/graphql` → v1. При breaking changes → `/api/graphql/v2`.
12 месяцев поддержки deprecated версии после анонса.

**GraphQL usage analytics** (`/admin/graphql/analytics`):
Топ дорогих запросов, медленных полей, пользователей с highest complexity.

### 5. Federated Data View

Реализация схемы `organization_regions` из v3.0.

```sql
-- Реализация (схема заложена в v3.0)
INSERT INTO organization_regions (org_id, region, is_primary)
VALUES ($orgId, $dataRegion, TRUE);

-- Агрегаты из обоих регионов (materialized)
CREATE TABLE org_federated_stats (
  org_id          UUID NOT NULL REFERENCES organizations(id),
  period          DATE NOT NULL,
  region          TEXT NOT NULL CHECK (region IN ('eu', 'us')),
  total_cost_usd  NUMERIC(10,6) DEFAULT 0,
  request_count   INTEGER DEFAULT 0,
  active_members  INTEGER DEFAULT 0,
  p95_latency_ms  NUMERIC,
  updated_at      TIMESTAMPTZ DEFAULT now(),
  PRIMARY KEY (org_id, period, region)
);
```

**Materialized Federation:**
Inngest job каждые 15 минут:
1. Читает `daily_usage_rollups` из EU Data Plane
2. Читает `daily_usage_rollups` из US Data Plane
3. Агрегирует → `UPSERT INTO org_federated_stats`

Dashboard `/org/{id}/global-overview` читает только из `org_federated_stats` —
нет кросс-региональных запросов в runtime.

**Экспорт:**
`GET /api/org/{id}/federated-stats?from=2026-01-01&to=2026-07-01&region=all`
→ CSV или JSON с полями: `period`, `region`, `total_cost_usd`, `request_count`.

### 6. ML-прогнозы расходов

```sql
ALTER TABLE daily_usage_rollups
  ADD COLUMN predicted_cost_next_7d     NUMERIC(10,6),
  ADD COLUMN prediction_lower_bound     NUMERIC(10,6),  -- conformal prediction
  ADD COLUMN prediction_upper_bound     NUMERIC(10,6),
  ADD COLUMN prediction_confidence      NUMERIC(4,3),   -- 0.0 - 1.0
  ADD COLUMN prediction_explanation     JSONB,
    -- { "trend_pct": 10.5, "seasonality": null, "data_points": 14 }
  ADD COLUMN prediction_cohort_used     BOOLEAN DEFAULT FALSE,
  ADD COLUMN prediction_updated_at      TIMESTAMPTZ;
```

**Cold Start через когорту:**
```typescript
const MIN_DATA_POINTS = 14; // минимум 2 недели для персональной модели

if (dataPoints < MIN_DATA_POINTS) {
  // Когорта: org того же plan_type в первые N дней
  const cohortAvg = await getCohortAverage(org.planType, dataPoints);
  return { ...cohortAvg, prediction_cohort_used: true };
}
// Персональная линейная регрессия с seasonal decomposition
```

**Conformal Prediction (confidence interval):**
```typescript
// Вместо point estimate → интервал [lower_bound, upper_bound]
// Алерт только при upper_bound > 0.8 * monthly_budget_usd
// И prediction_confidence > 0.8
if (upperBound > monthlyBudget * 0.8 && confidence > 0.8) {
  await createBudgetAlert(orgId, { upperBound, confidence, explanation });
}
```

**Объяснимость прогноза в UI:**
```
"Прогноз: $45–62 за 7 дней (confidence 87%)
 Факторы: тренд +10.5% за 14 дней, сезонность не обнаружена
 Основан на 14 точках данных (персональная модель)"
```

**Рекомендации при превышении порога:**
```
"Прогноз превышает 80% бюджета ($80/$100).
 Рекомендации:
 • Перейти с gpt-4o на gpt-4o-mini для запросов без reasoning (-40% стоимость)
 • Установить лимиты на участников (текущий топ: user@co.com $30)
 • Включить auto-recharge или увеличить бюджет"
```

**Ежемесячная метрика точности:**
```sql
CREATE TABLE prediction_accuracy_log (
  org_id          UUID NOT NULL,
  period          DATE NOT NULL,
  predicted_cost  NUMERIC(10,6),
  actual_cost     NUMERIC(10,6),
  mape            NUMERIC(6,4),  -- Mean Absolute Percentage Error
  PRIMARY KEY (org_id, period)
);
```

При MAPE > 30% за 3 месяца подряд → алерт дата-команде платформы.

### 7. Marketplace v2

**Rating System:**
```sql
CREATE TABLE marketplace_provider_reviews (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id          UUID NOT NULL REFERENCES organizations(id),
  provider_id     UUID NOT NULL REFERENCES marketplace_providers(id),
  rating          INTEGER NOT NULL CHECK (rating BETWEEN 1 AND 5),
  comment         TEXT,
  verified_usage  BOOLEAN DEFAULT FALSE,
  total_spent_usd NUMERIC(10,4),        -- на момент отзыва
  org_plan_type   TEXT,                 -- для weighted rating
  moderation_status TEXT DEFAULT 'pending'
                  CHECK (moderation_status IN ('pending', 'approved', 'rejected', 'suspicious')),
  moderated_by    UUID,
  created_at      TIMESTAMPTZ DEFAULT now(),
  UNIQUE (org_id, provider_id)
);
```

**Защита от накрутки:**
- `verified_usage = TRUE` только если org потратила >$10 у провайдера
- Weighted rating: Enterprise план — вес 3x, Pro — 1.5x, Free — 1x
- Аномалия-детектор (Inngest job): если >20 отзывов провайдеру за 24 часа → `moderation_status = 'suspicious'` + алерт
- Auto-moderation: спам-фильтр + нормативная лексика перед сохранением

**Версионирование моделей:**
```sql
ALTER TABLE marketplace_providers ADD COLUMN
  model_versions JSONB DEFAULT '[]';
  -- [{ "id": "command-r", "version": "2024-08", "deprecated_at": null, "auto_switch": true }]
```

Grace period **opt-out** (не opt-in):
- За 60 дней: email всем org, которые используют модель → `Модель будет обновлена. Отключить auto-switch?`
- За 30 дней: повторное уведомление
- Org с `auto_switch = FALSE` могут pin specific version (surcharge $X/month)
- Без pin: автопереключение на `latest` в день deprecation

**Provider notifications:**
При добавлении новой версии модели → email провайдеру с deadline обновления параметров.
При `deprecated_at IS NOT NULL` → email всем connected org с планом миграции.

**Provider analytics (GDPR-compliant):**
Dashboard только с агрегированными данными: total requests (платформа), avg latency,
error rate. Без разбивки по org или пользователям.

### 9. API Versioning Strategy

**REST:** URL versioning. Текущий: `/api/v1/`. При breaking changes → `/api/v2/`.
12 месяцев поддержки после deprecated announcement с `Deprecation: date` заголовком.

**GraphQL:** Schema versioning. Breaking changes → новая версия схемы.
Field-level deprecation через `@deprecated(reason: "...")`.

**Changelog:** `GET /api/changelog` — машиночитаемый список изменений по версиям.

**SDK versioning:** Semantic versioning. Breaking changes только в major versions.
`@deprecated` аннотации за 2 minor versions до удаления метода.

## Definition of Done

- [ ] Dry-run миграции проходит и автоочищается без следов в target регионе
- [ ] CDC: изменения во время migration_changes_log применяются корректно
- [ ] Checksum mismatch при миграции → автоматический rollback
- [ ] Без 2FA подписи обоих суперадминов → миграция не запускается
- [ ] `trace_token` одноразовый: повторное использование → 401
- [ ] `trace_token` с истёкшим TTL (30 сек) → 401
- [ ] 10 одновременных callbacks с одним `event_id` → ровно одна запись в `byok_callbacks`
- [ ] Без callback через 5 мин → резерв освобождён, алерт создан
- [ ] GraphQL запрос с complexity > limit → 400 с указанием score
- [ ] Произвольный query string в production → 400 Bad Request
- [ ] `organization { members { usageEvents } }` не генерирует N+1 (DataLoader)
- [ ] Mobile SDK: rotation SSL сертификата не требует обновления SDK (Public Key Pinning)
- [ ] Apollo Codegen: Swift/Kotlin типы полностью соответствуют GraphQL схеме
- [ ] Для org < 14 дней: прогноз из когорты, `prediction_cohort_used = TRUE`
- [ ] Алерт бюджета только при upper_bound > 80% И confidence > 80%
- [ ] Отзыв без `verified_usage = TRUE` → 403
- [ ] 25 отзывов провайдеру за 24 часа → `moderation_status = 'suspicious'` + алерт
- [ ] Grace period opt-out: org с `auto_switch = FALSE` не переключается автоматически
- [ ] Standby synthetic test failure → алерт до реального инцидента
- [ ] Gradual failover: 5% трафика перед 100% переключением
- [ ] Admin org получает email/Slack при активации Read-Only
- [ ] Game Day (Chaos Engineering): EU staging отключается → Standby активируется за < 2 мин
- [ ] npm run verify — green (EU и US инстансы)
- [ ] Smoke-test на production
