# 10 - UI Pages

## Назначение файла

Этот файл описывает страницы, экраны, компоненты и правила интерфейса проекта **Новая эпоха**.

Интерфейс проекта должен помогать пользователю выполнить главный сценарий:

```text
Ввести задачу.
# пользователь формулирует prompt

Выбрать несколько AI-моделей.
# пользователь выбирает участников сравнения

Получить ответы рядом.
# система показывает результаты в удобном виде

Выбрать лучший ответ.
# в v0.3 это локальный UI-выбор, в v0.6 это сохранённый vote

Сохранить сравнение.
# появляется после Supabase и Voting MVP
```

Главная цель интерфейса - не создать обычный чат, а сделать удобную платформу для сравнения AI-моделей.

## Главный источник порядка версий

Порядок появления страниц должен соответствовать файлу:

```text
14-roadmap.md
# главный источник порядка версий проекта
```

Если этот файл конфликтует с `14-roadmap.md`, правильным считается порядок из `14-roadmap.md`.

# Общий принцип интерфейса

Проект **Новая эпоха** не должен выглядеть как один обычный AI-чат.

Обычный AI-чат:

```text
Один запрос -> одна модель -> один ответ
```

Проект **Новая эпоха**:

```text
Один запрос -> несколько моделей -> несколько ответов -> сравнение -> выбор лучшего -> история -> рейтинг
```

Поэтому основной экран должен быть построен вокруг сравнения ответов, а не вокруг длинной переписки с одной моделью.

# Что не входит в MVP

В MVP не входят:

- Code Arena Runner;
- AI Team Mode;
- сложная админ-панель;
- платёжная система;
- публичные профили;
- сложная аналитика;
- запуск чужого кода;
- командные цепочки моделей;
- продвинутый Leaderboard;
- полноценная система тарифов.

Эти функции добавляются позже по `14-roadmap.md`.

# Страница Prompt Arena

## Путь

```text
/arena
# главная рабочая страница MVP
```

## Версии развития

```text
v0.3
# UI с mock-ответами и локальным выбором победителя

v0.4
# реальные ответы через backend route

v0.5
# сохранение задач и ответов в Supabase

v0.6
# сохранение выбора лучшего ответа

v1.0
# стабильная Prompt Arena
```

## Назначение

Prompt Arena - главный экран MVP. Здесь пользователь вводит задачу, выбирает 2 или 3 модели, получает ответы и выбирает лучший вариант.

## Основной сценарий v0.3

```text
Пользователь вводит prompt.
# текст задачи

Пользователь выбирает 2 или 3 mock-модели.
# текущий UI MVP не подключает реальные модели

Пользователь нажимает кнопку запуска сравнения.
# UI запускает mock-генерацию

UI показывает ответы рядом.
# каждая модель получает отдельную карточку

Пользователь выбирает победителя.
# выбор локальный и не сохраняется после обновления страницы
```

## Основной сценарий v1.0

```text
Пользователь вводит prompt.
# текст задачи

Пользователь выбирает 2 или 3 разрешённые модели.
# модели берутся из allowlist/backend

Frontend отправляет запрос на backend.
# внешний AI API не вызывается из браузера напрямую

Backend возвращает ответы.
# success и error отображаются отдельными карточками

Пользователь выбирает победителя.
# с v0.6 выбор сохраняется через /api/vote

Сравнение можно открыть позже в истории.
# появляется с v0.7
```

# Prompt input

Минимальные требования:

- многострочный ввод;
- placeholder с примером;
- нельзя отправлять пустой prompt;
- минимум 3 символа;
- максимум 8000 символов;
- textarea должен иметь `maxLength=8000`;
- frontend должен показывать понятную ошибку;
- backend в будущем обязан повторно проверять лимит.

---

# Model selector

В MVP пользователь должен выбрать минимум 2 модели.

Правила:

```text
Минимум 2 модели
# иначе сравнение не имеет смысла

Максимум 3 модели в MVP
# чтобы не тратить слишком много токенов и не перегружать UI

Только разрешённые модели
# список должен приходить из allowlist/backend на следующих этапах
```

Карточка модели должна показывать:

- название;
- provider;
- краткое назначение;
- badge или price label, если доступно.

---

# Run button

Кнопка запуска должна быть неактивной или возвращать ошибку, если:

- prompt пустой;
- prompt короче 3 символов;
- prompt длиннее 8000 символов;
- выбрано меньше 2 моделей;
- выбрано больше 3 моделей;
- уже идёт генерация;
- backend временно недоступен на будущих этапах.

Текущий текст кнопки в UI MVP:

```text
Запустить сравнение
# запуск mock-сравнения
```

Во время загрузки:

```text
Генерация mock-ответов...
# ответы генерируются локально
```

---

# Response area

Ответы моделей должны отображаться рядом.

На desktop:

```text
2 модели -> 2 карточки
3 модели -> 3 карточки или адаптивная сетка
```

На mobile:

```text
1 колонка
# карточки идут одна под другой
```

---

# ResponseCard

Каждая карточка ответа должна содержать:

- название модели;
- роль модели в UI;
- статус ответа;
- `answerText`, если ответ успешный;
- `errorMessage`, если модель вернула ошибку;
- `errorCode`, если доступен;
- время генерации, если доступно;
- кнопку выбора победителя только для успешного ответа.

Статусы:

```text
success
# ответ получен

error
# модель не ответила или вернула ошибку
```

# UI winner selection и Voting MVP

Нужно различать два этапа.

```text
v0.3 - UI winner selection
# пользователь может визуально выбрать победителя в текущем состоянии страницы
# выбор не сохраняется в базе

v0.6 - Voting MVP
# выбор сохраняется через /api/vote и таблицу votes
```

Текущий текст кнопки в UI MVP:

```text
Выбрать лучшим
# локально выбрать карточку победителем
```

После выбора:

```text
Выбрано лучшим
# карточка отмечена в UI
```

# Ошибки на Arena

UI должен показывать понятные ошибки:

```text
Введите задачу минимум из 3 символов.
# prompt слишком короткий

Введите задачу не длиннее 8000 символов.
# prompt слишком длинный

Для сравнения нужно выбрать минимум две модели.
# выбрано меньше двух моделей

В MVP можно выбрать максимум три модели.
# выбрано больше лимита MVP

Модель не вернула ответ.
# fallback для карточки ответа
```

На будущих backend-этапах также нужны ошибки:

```text
Model did not respond
# одна модель не вернула ответ

Server error
# ошибка backend

Rate limit exceeded
# превышен лимит запросов
```

# Что нельзя делать в Prompt Arena MVP

- не делать полноценный чат с историей сообщений;
- не добавлять сложные цепочки агентов;
- не запускать код;
- не делать AI Team Mode внутри Arena;
- не добавлять оплату;
- не давать пользователю вводить произвольный model ID;
- не показывать server-side переменные в UI.

# Страница Code Arena Lite

## Путь

```text
/code
# страница Code Arena Lite
```

## Версия

```text
v1.1
# после Stable Prompt Arena
```

## Назначение

Code Arena Lite сравнивает ответы моделей по задачам программирования, но не запускает код.

Что можно делать:

- пользователь вводит задачу по коду;
- пользователь может вставить фрагмент кода;
- модели предлагают решение;
- ответы сравниваются рядом;
- пользователь выбирает лучший вариант.

Что нельзя делать в Lite:

```text
Не запускать код.
# запуск чужого кода запрещён до v1.7

Не использовать sandbox.
# sandbox появится в Code Arena Runner

Не создавать code_results.
# результаты запуска появятся только в v1.7
```
