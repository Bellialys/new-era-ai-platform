# 02 - План проекта

## Назначение файла

Этот файл описывает общий поэтапный план разработки проекта **Новая эпоха**.

Главная задача файла - показать, как проект должен двигаться от документации и простого интерфейса к стабильной AI-платформе сравнения моделей.

Главный источник порядка версий - `14-roadmap.md`.

Если этот файл конфликтует с `14-roadmap.md`, правильным считается `14-roadmap.md`.

---

# Главный принцип разработки

Проект разрабатывается без хаоса и без попытки сразу создать большую систему.

Основные правила:

1. Сначала создаётся понятная документация.
2. Потом создаётся базовая Next.js основа.
3. Потом делается Static UI MVP для Prompt Arena.
4. После этого подключаются OpenRouter API, Supabase, голосование и история.
5. Только после стабильного Prompt Arena добавляются Code Arena, Multi Model Battle, Judge Mode и Leaderboard.
6. Code Arena Runner и AI Team Mode добавляются только после стабильной базы, лимитов и контроля безопасности.
7. Каждый этап должен давать рабочий результат.
8. После каждого важного этапа нужно делать Git commit.
9. Новая функция не должна ломать уже работающую часть проекта.

# Что считается MVP

В проекте есть два уровня MVP.

## 1. UI MVP

UI MVP - это версия `v0.3`.

Она показывает основную идею проекта без реальных AI-запросов и без базы данных.

Входит:

- главная страница;
- страница `/arena`;
- поле ввода задачи;
- выбор 2-3 mock-моделей;
- кнопка запуска сравнения;
- loading state;
- empty state;
- error state;
- mock-ответы;
- локальный UI-выбор победителя.

Не входит:

- OpenRouter API;
- Supabase;
- сохранение задач;
- сохранение ответов;
- сохранение голосов;
- история;
- production deploy.

## 2. Stable Prompt Arena MVP

Stable Prompt Arena MVP - это версия `v1.0`.

Она должна доказать главную ценность проекта: пользователь отправляет одну задачу нескольким AI-моделям, получает ответы рядом, сравнивает их, выбирает лучший вариант и может открыть результат позже.

Входит:

- реальная отправка запроса через OpenRouter API;
- получение ответов от нескольких моделей;
- отображение ответов рядом;
- сохранение задачи и ответов в Supabase;
- сохранение голоса через `/api/vote`;
- история запросов;
- базовая обработка ошибок;
- базовые лимиты;
- деплой на Vercel.

# Фаза 5 - Supabase Integration / v0.5

## Цель

Сохранять модели, задачи и ответы в Supabase PostgreSQL.

## Что нужно сделать

- создать проект Supabase;
- создать таблицы MVP;
- подключить Supabase client;
- добавить переменные окружения;
- сохранять задачи пользователя;
- сохранять ответы моделей;
- сохранять ошибки моделей;
- подготовить таблицу `votes` для следующей фазы.

## Минимальные таблицы

- `models` - список доступных моделей;
- `tasks` - задачи пользователей;
- `model_responses` - ответы моделей;
- `votes` - голоса за лучший ответ.

## Результат

Запросы и ответы больше не теряются после обновления страницы.

# Что не входит в ранний MVP

До `v1.0` не добавляем:

- регистрацию пользователей;
- оплату;
- сложный Leaderboard;
- AI Team Mode;
- запуск чужого кода;
- полноценный sandbox;
- сложную админ-панель;
- расширенную аналитику;
- командную работу моделей;
- автоматическое тестирование кода;
- сложные тарифы и подписки.

Эти функции добавляются позже, когда базовая Prompt Arena уже работает стабильно.
