## Постоянные правила сессии (для агента-исполнителя)

Ты работаешь с репозиторием `https://github.com/Bellialys/new-era-ai-platform` (Next.js / TypeScript / Supabase / Vercel). Правила обязательны для каждой задачи без исключений:

1. **Свежая база.** Перед началом: `git fetch origin && git checkout main && git pull`. Каждая задача — новая ветка `fix/task-<N>-<короткое-имя>` от актуального main.
2. **Никогда не пушить в main.** Результат каждой задачи — Pull Request: `gh pr create --title "..." --body "..."`. В body PR: номер задачи из спеки, список изменённых файлов, вывод финального прогона проверок.
3. **Definition of Done** (прогонять локально, PR открывать только при полностью зелёном результате):
   ```
   npm run typecheck && npm run lint && npm run test
   ```
   Node-версия — из `.nvmrc` (24). Если тесты падают — чинить и перезапускать до зелёного, а не открывать PR с оговорками.
4. **Документация в том же коммите, что и код** (политика `36-document-sync-policy.md`): изменения API → `28-api-contracts.md`; схема БД → `08-database.md`; каждая задача → строка в `15-changelog.md` под текущей версией.
5. **Миграции НЕ применять.** Если задача включает миграцию: создать SQL-файл в `supabase/migrations/` с временным timestamp, включить в PR и написать в body PR строку: `⚠️ MIGRATION PENDING — применяет Claude (web) через Supabase MCP после merge`. Никаких `supabase db push`, никаких прямых подключений к базе. После применения миграции владелец пришлёт отдельную микрозадачу на `git mv` файла под фактический timestamp.
6. **Секреты.** Не создавать и не коммитить `.env*` (кроме правок `.env.example`), не вписывать ключи в код, тесты и документацию. Репозиторий публичный.
7. **Границы задач соблюдать буквально.** Каждая задача в `46-p0-fix-specs.md` имеет раздел «Явные границы» / «Не трогать» — выход за скоуп запрещён, даже если видишь что улучшить. Замечания вне скоупа — отдельным списком в body PR, не в коде.
8. **Стиль ошибок и лимитов** — только паттерны проекта: `ApiError` + `createErrorResponse` (форма ответа плоская: `{ status, errorCode, message }`), `logApiRequest` на каждом выходе, 429 всегда с `Retry-After`. Эталон: `src/app/api/vote/route.ts` и его тест.
