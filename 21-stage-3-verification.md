# Что реализовано в UI

## Форма

- Поле ввода prompt.
- Выбор моделей через checkbox.
- Счётчик выбранных моделей.
- Кнопка запуска сравнения.
- Кнопка очистки.

## Валидация

- Prompt должен быть минимум 3 символа.
- Нужно выбрать минимум 2 модели.
- В MVP можно выбрать максимум 3 модели.
- Ошибка показывается в интерфейсе.

## Состояния

- Empty state - ответов ещё нет.
- Loading state - имитация генерации ответов.
- Error state - ошибка валидации.
- Success state - карточки mock-ответов.

## Ответы моделей

- Ответы появляются после нажатия кнопки.
- У каждого ответа есть модель, роль, статус и latency.
- Текст ответа зависит от prompt пользователя.
- Можно выбрать лучший ответ.
- Победитель получает визуальную метку `Winner`.

# Ручной тест в браузере

После запуска `npm run dev` открыть:

```text
http://localhost:3000/arena
# страница Prompt Arena
```

Проверить сценарии:

1. Открыть `/arena`.
2. Нажать запуск с пустым prompt.
3. Убедиться, что появляется ошибка.
4. Ввести задачу меньше 3 символов.
5. Убедиться, что появляется ошибка.
6. Оставить выбранной только одну модель.
7. Убедиться, что появляется ошибка.
8. Ввести нормальный prompt.
9. Выбрать 2 или 3 модели.
10. Нажать запуск сравнения.
11. Убедиться, что появляется loading state.
12. Убедиться, что после загрузки появляются mock-ответы.
13. Нажать `Выбрать лучшим`.
14. Убедиться, что выбранная карточка получает метку `Winner`.
15. Нажать `Очистить`.
16. Убедиться, что форма и ответы сбрасываются.
