# 21 - Access Gate Policy

## Назначение файла

Этот документ фиксирует обязательное правило доступа для проекта **Новая эпоха**.

Пользователь не должен использовать рабочие функции платформы как полностью неизвестный посетитель. Перед запуском Prompt Arena он должен выбрать один из двух режимов:

1. войти или зарегистрироваться;
2. продолжить как гость и получить anonymous session с карточкой `Анонимус #1234`.

## Главное правило

```text
Prompt Arena нельзя использовать без user session или anonymous session.
# либо аккаунт, либо гостевая карточка
```

## Что можно без авторизации

Без аккаунта и без гостевой карточки можно:

- открыть главную страницу;
- открыть публичный лендинг;
- открыть `/auth`;
- открыть `/api/health`;
- прочитать описание проекта;
- нажать `Продолжить как гость`.

## Что нельзя без user/guest session

Без аккаунта и без гостевой карточки нельзя:

- запускать `/api/compare`;
- выбирать модели для реального запуска;
- отправлять prompt;
- сохранять task;
- голосовать;
- смотреть личную историю;
- использовать любые registered/premium модели.

## UI-правило

Пока пользователь не вошёл и не выбрал гостевой режим:

- поле prompt должно быть disabled;
- выбор моделей должен быть disabled;
- кнопка запуска должна быть disabled;
- вместо формы или над формой должен быть Access Gate block.

Текст Access Gate:

```text
Чтобы начать сравнение моделей, войдите в аккаунт или продолжите как гость.

[Войти]
[Создать аккаунт]
[Продолжить как гость]
```

## Guest Mode

При нажатии `Продолжить как гость` система должна:

1. создать или восстановить guest session через backend route `POST /api/guest`;
2. создать карточку вида `Анонимус #4827`;
3. сохранить display-информацию гостя в localStorage;
4. показать guest card в интерфейсе;
5. разрешить только бесплатные модели;
6. отправлять `/api/compare` без guest id в body; backend использует httpOnly cookie `na_guest`.

## Backend-правило

Frontend-блокировка недостаточна. Backend обязан проверять доступ.

`/api/compare` должен:

- определить авторизованного пользователя через Supabase Auth, если он есть;
- если пользователя нет, проверить httpOnly cookie `na_guest`;
- если нет ни пользователя, ни валидной гостевой сессии - вернуть `401 AUTH_REQUIRED`;
- если гость выбирает модель не для anonymous-доступа - вернуть `403 MODEL_NOT_ALLOWED`;
- для аккаунта сохранять `tasks.user_id`;
- для гостя сохранять `tasks.anonymous_session_id`.

## Model access levels

Модели должны иметь уровень доступа:

```text
anonymous
# доступно гостям и аккаунтам

registered
# доступно только аккаунтам

premium
# будущий тарифный доступ
```

Гость видит и использует только `anonymous` модели.

## Критерий готовности

Готово, если:

- без входа и без гостевой карточки нельзя отправить prompt;
- `/api/compare` возвращает `401 AUTH_REQUIRED`, если нет user/guest session;
- при выборе гостя появляется карточка `Анонимус #xxxx`;
- гостевая карточка сохраняется после перезагрузки;
- guest mode создаёт `na_guest` cookie через backend и не передаёт guest id в request body;
- гость использует только бесплатные модели;
- `npm run typecheck`, `npm run lint`, `npm run build`, `npm run smoke` проходят без ошибок.
